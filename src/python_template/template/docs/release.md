# Release

This project is published to PyPI as `{{project_name}}`. The import package is
`{{package_name}}`.

## Before Publishing

Confirm the package name is available or owned by you on PyPI and TestPyPI:

- <https://pypi.org/project/{{project_name}}/>
- <https://test.pypi.org/project/{{project_name}}/>

Run the local release checks:

```bash
nox -s test
nox -s lint
nox -s docs
nox -s build
```

## Versioning

Versions are generated by `setuptools-scm` from Git tags. Create release tags
with a `v` prefix:

```bash
git tag v0.1.0
git push origin v0.1.0
```

## Publishing

The GitHub workflow `.github/workflows/publish-pypi.yml` supports two flows:

- Run the workflow manually to publish to TestPyPI.
- Publish a GitHub release to publish to PyPI.

The workflow currently expects these repository secrets:

- `TEST_PYPI_TOKEN`
- `PYPI_API_TOKEN`

You can also configure PyPI trusted publishing for this GitHub repository and
remove token-based secrets from the workflow later.
